# Report Format Conventions

## File Naming
Reports are saved to `outputs/reports/` with format: `{brand.name.lower()}_YYYY-WNN.md`
Example: `yourbrand_2026-W09.md` (using the name from `config/brand.yaml`)

Week number uses ISO 8601 (Monday = start of week). Use Python-style: `datetime.now().strftime("%Y-W%V")` equivalent.

---

## Inline Summary Format (terminal output, not saved to file)

When presenting results in the terminal (after `/analyze-reviews`), use this condensed format:

```
PAPIER FEEDBACK ANALYSIS — YYYY-WNN
────────────────────────────────────
Reviews: N | Positive: N% | Neutral: N% | Negative: N%

TOP THEMES
  1. delivery (N mentions)
  2. ...

HIGH-PRIORITY ISSUES (score ≥ 6.0)
  ★1 synth_XXX — "[excerpt]" (score: N.N)
  ...

KEY INSIGHTS
  Opportunities: [one-liner for each]
  Risks: [one-liner for each]

Run /generate-report to save full report to outputs/reports/
```
